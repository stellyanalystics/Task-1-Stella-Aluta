SQL Internship Project – Online Retail Data Analysis
 Project Overview
This is my first project as an intern with Decodelabs tech. The project focuses on data cleaning, exploratory data analysis (EDA), and SQL data analysis of a retail sales dataset using SQL and Excel. The objective is to understand sales patterns, customer behavior, and product performance through structured data analysis.

Tools Used 
•	SQL Server Management Studio (SSMS) 
•	Excel / WPS Spreadsheet 
•	GitHub 

 Project Structure
•	Week 1: Data Cleaning 
•	Week 2: Exploratory Data Analysis (EDA) 
•	Week 3: SQL Data Analysis 
 
 Week 1 – Data Cleaning
In this stage, the dataset was cleaned to improve accuracy and consistency by:
•	Removing duplicates 
•	Handling missing values (NULLs) 
•	Standardizing data types
•	Verifying data consistency 

Week 2 – Exploratory Data Analysis (EDA)
Key Analyses Performed
•	Analyzed the dataset using basic statistics to understand patterns and distributions. 
•	Identified sales trends over time. 
•	Calculated total revenue. 
•	Evaluated product performance using transaction counts. 
•	Identified outliers and unusual sales patterns. 

 Week 3 – SQL Data Analysis
SQL Skills Demonstrated
•	SELECT queries 
•	WHERE filtering 
•	ORDER BY sorting 
•	GROUP BY aggregation 
•	COUNT, SUM, and AVG functions 
•	Data type conversion and cleaning 
                     
Key Insights (EDA Insights)
•	Sales data is generally stable, with only one identified outlier. 
•	The average transaction value is influenced by a small number of high-value purchases. 
•	Weekend periods, especially Sundays, generate the highest sales revenue and quantity sold. 
•	Mid-year months demonstrate stronger sales performance compared to other periods. 
•	Customer demand appears consistent, indicating a stable and predictable sales environment. 

SQL-Based Insights
•	High-value transactions were identified using filtering conditions (UnitPrice > 500). 
•	Product-level analysis shows that certain products contribute significantly more to total revenue than others. 
•	Revenue distribution is uneven, with a small number of products generating the majority of sales. 
•	Order frequency analysis using COUNT() revealed variations in product demand. 
•	Average order value analysis using AVG() provided insight into typical customer spending behavior. 

Recommendations
•	Increase marketing efforts during high-performing months and weekends. 
•	Investigate high-value transactions to identify opportunities for premium product offerings. 
•	Maintain sufficient inventory levels before weekends due to increased demand. 
•	Monitor low-performing days and introduce targeted promotional campaigns. 
•	Continue tracking sales trends to improve forecasting and inventory planning. 

Conclusion
The analysis reveals a stable retail environment with predictable customer behavior and clear sales patterns. A small number of products and specific time periods contribute disproportionately to revenue, highlighting opportunities for targeted marketing, improved inventory management, and data-driven decision-making.
